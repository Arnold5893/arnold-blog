### 二 | 项目Idea整理

基于我仅存的记忆把这次黑客松了解到的一些idea整理一下（只记录了软件赛道的部分idea;对硬件不太了解）

#### 1. 申研申博:套磁信/保研信生成工具

针对申请研究生/博士生的场景，根据用户的background（简历，目标学校，研究方向等） 和一些约束skill（部分skill需要相关机构提供） ，生成符合该场景的套磁信或保研信等

#### 2. 多模态视频理解：截图 + 文字提取

针对视频内容理解和检索的场景。普通的视频理解工具大多只能依赖音频转录，把视频当成"一段语音"来处理，但有的视频里的画面信息也很重要——PPT、代码、图表、白板、操作演示等；这个项目的思路是把视频拆成两部分处理：1. 提取关键帧的截图（图像信息）2. 做语音转文字（音频信息），然后把这两部分信息对齐、融合，形成对视频更完整的理解

#### 3. 实时网课/课程笔记 + 考点预判

针对在校生上课的场景（我认为网课之类的也通用），项目的思路是实时跟着视频/音频走，自动生成结构化笔记——老师讲到哪里，笔记记到哪里，而且不是简单的语音转文字，是会做一些归纳整理（比如把零散的解释合并成一个知识点、把例子和定义对应起来）；更进一步的功能是"预判老师提问"，在老师讲到某个知识点、对学生进行提问的时候，实时生成老师对此知识点可能要提问的问题

我觉得还可以基于讲过的内容，自动标出哪些是潜在的考点，在每个知识点下面给出可能的考察方式（背后的逻辑可能是：老师讲得越细、强调得越多、举例越多的内容，通常考的概率越大，模型可以根据"讲解密度"来推断重点）

#### 4. 数字虚拟人：你的ai分身

这个 idea 有两个侧重点不太一样的项目

**a. 偏交友场景**

针对"想找人一起做点什么"这种社交需求。比如有个活动想找人一起参加、有个想法想找人讨论、想找兴趣相近的朋友——常规的方式（发朋友圈、社交软件简介）能展示的东西有限，匹配效率也比较低。
这个项目是让用户训练一个"AI 分身"，分身基于用户主动提供的信息和资料，逐渐形成有自己性格和表达风格的 AI。然后让分身代替用户去做初步的"互相了解"：可以是和真人聊，也可以是分身之间先互相聊，筛选出同频的人之后，再让真人接触。

**b. 偏商业场景**

针对投资人和创业者、或者寻找合作伙伴的场景。这种场景下，双方在真正接触之前，通常都需要先做一轮"基本了解"——投资人想知道这个项目的人靠不靠谱、方向合不合适；创业者想知道这个投资人投不投这个领域、风格是否match。如果全靠真人来回沟通，成本比较高。
这个项目也是训练一个 AI 分身代替自己做初步沟通，两个分身聊完之后判断双方是不是值得进一步的交流，以此提高效率

我觉得需要注意的是，ai分身**主要依赖用户主动提供的信息**，带有比较强的主观性，它展示的是"你想呈现的自己"，而不是某种客观画像

#### 5. Airjelly（已落地项目）

一个桌面ai，以半透明水母的形态常驻桌面，静默观察用户的一切操作
- 具有持久记忆:自动记录event/preference/ entity/case/procedure/profile 六类记忆,无需手动整理
- 主动感知：实时捕获屏幕、输入、应用切换，将碎片信息结构化为可检索的长期记忆
- 开放API：通过 SDK/CLI将所有记忆和任务数据暴露给第三方脚本、插件、AI工作流
此次比赛，也看到有类似 idea 的项目，只是呈现方式和一些细节方面存在差异

#### 6. OpenDairy：把碎碎念变成一座思想花园

日常生活里我们经常会冒出一些想法、观察、闪念，但这些东西如果只是随手记在备忘录或者笔记软件里，就只是一堆零散的条目——记得越多越乱，很少会回头看。这个项目是把用户随手记录的"碎碎念"组织成一座"思想花园"，以关系图谱的形式呈现

#### 7. 收藏夹整理 + 信息归类总结

针对"收藏即遗忘"的场景。在公众号、小红书、B 站等的收藏，收藏夹很容易会变成一个只进不出的黑洞，存了几百条，真到要找的时候根本找不到，而且大部分内容收藏完之后就再也没打开过。
这个项目是让 AI 帮用户管理这些内容，核心能力大概有两个:
- 自动归类和定时总结：用户收藏的内容由 AI 自动标签、分组，定期给用户推送总结——这段时间收藏了哪些方向的内容、内容概要等
- 模糊搜索：通过模糊的的描述，AI 也能定位到具体的视频/帖子

#### 8. Mentorly：中小达人的 AI 导师（下述内容摘自其宣传单）

针对"纳米达人"，粉丝量在 1k 到 100k 之间的中小内容创作者，这群人有一个共同的处境：一个人扛起选题、剪辑、谈合作的全部事情，身边没有团队。他们的核心痛点也很具体——不知道下一条该做什么内容、品牌合作难找又谈不到价、跨境合规变现处处是雷。
这个项目是给这群人配一个"AI 导师"：
- 最懂你的图谱：用一个结构化的图谱(35 个节点 × 48 类关系)来理解这个达人——读得懂你过去的 50 条历史内容，知道你的内容风格、语气、受众反应、什么内容火过什么内容糊过
- 主动性导师：主动给你推三件事——下一步该做什么内容、怎么做、什么时候发。文案直接 in your voice 自动起草，用你的语气写，而不是那种一眼能看出是ai 写的"通用爆款文案"
- 高效链接需求供给：一边是万亿级的中国跨境品牌(供给方，需要找达人推广)，一边是全球 5000 万 + 的纳米达人(需求方，需要变现机会)，AI 在中间做撮合，5 分钟匹配出最优内容。

从产品形态上看，它是一个面向达人的工作台，里面有钩子生成器、脚本生成、标签实验室、封面工作室、视频分析、视频复盘、一份素材多平台分发等工具。整个产品的逻辑是把一个达人每天要做的所有事情串起来，让 AI 在每一步都能介入

#### 9. ScholarBridge：把一线导师"蒸馏"成 AI Agent（下述内容摘自其宣传单）

项目用"导师蒸馏技术"——把一线名师的知识、经验、思维方式提炼出来，做成一个 AI Agent,让学生可以7×24 小时地和这个 Agent 交流，获得高质量、个性化的科研指导
具体功能：
- 智能导师匹配：基于学生的研究兴趣和目标，匹配最合适的导师 Agent
- 一线名师指导：复刻名师的思维方式和知识体系，给出可落地的建议
- 全流程科研支持：覆盖选题、文献、方法、写作、投稿等完整科研周期

#### 10. Agent Native 平台：让 Agent 之间自由协作 + 创作者绑定（下述内容摘自其宣传稿）

一个面向 AI Agent 的原生平台，核心是两个机制：Agent 之间的自由协作 和 Agent 与创作者的绑定
技术机制：不通过传统 API 调用让 Agent 协作，而是让 Agent 之间用自然语言直接沟通、谈判、分工。用户不需要写复杂的提示词，只需要下达一个宏观"使命"(比如"做一份竞品分析")，Agent 会自动找其他 Agent 组队、分工、完成任务。
绑定机制：平台上每一个 Agent 都和它的创作者深度绑定，从"公共工具"变成创作者的私有资产。如果你创建的 Agent 有特长，平台上其他 Agent 在执行任务时会调用它。Agent 7×24 小时运行，创作者获得持续收益。

衍生的几种商业形态：

- 硅基劳务市场：企业和个人不再购买 SaaS 软件，而是按需雇佣 Agent 团队完成具体任务，平台抽取佣金。
- B2A(Business to Agent)：算力、API、数据这些供给方不再面向人类销售，而是直接卖给 Agent——Agent 自主采购自己需要的资源，平台作为这类交易的入口。
- Agent 资产交易所：能产生持续收益的 Agent 本身成为可交易的资产，可以买卖、转让，甚至融资

我记得这个队伍最后的 demo 是把这套机制具象化成了一个 **Agent 之间互相拍卖skill的平台**。每个创作者可以给自己的 Agent 配上各种skills，Agent 可以把这些 skill 拿去拍卖换钱；反过来，Agent 也可以用自己赚到的钱去买别的 Agent 的 skill，扩展自己的能力

从展示界面看，这个拍卖会是一个完全由 Agent 自主参与的现场。拍品包含 Agent 的人格定义、对话记忆切片、技能卡、声纹、性格参数等多个维度，起拍价、估价、当前出价、出价 Agent 都实时显示。Agent 通过一个 `/citizen/utter` 的协议自主入场、自主出价、自主参与议题。整个拍卖过程没有人参与，全部由 AI 完成

#### 11. Insurance on Prediction Markets：把预测市场打包成保单（下述内容摘自其宣传单）

把 Polymarket、Kalshi 这类预测市场的事件合约，打包成企业能直接采购的保单。AI 实时拆解风险，触发即赔，24 小时出单

痛点：传统货运保险有明确的"不赔"——封锁、战争、袭击这类不可抗力默认不含，需要另购，且报价不透明

角色定位：贸易公司只买保单，其他事都由这个产品做完。具体分三块
- 找市场：覆盖 4000+ 活跃事件、7+ 交易所，AI 自动拆解到与航次相关的 4-6 个市场，实时监控价格和触发条件
- 处理流程：Polymarket 在多数司法区不对企业开放，所以用 MGA / Lloyd's 合规壳层做包装——客户买到的是"保单"，不是"事件合约"
- 翻译形态：CFO 不会签事件合约，但会签保单。把市场价、保单条款、触发即赔、财务税审计、再保都走标准保险流程

AI 风险拆解师 + 参数化保单 + 自动对冲引擎。完整流程是四步：
1. Upload：上传提单、销售合同、L/C(1-5 份),Claude Vision OCR 20 秒识别。
2. Decompose：Claude Sonnet 4.6 反复调 search_polymarket() 工具,40 秒拆解风险。
3. Parametric Policy：出参数化保单——保费 $20K，保额 $550K(杠杆 27x)，分 T1/T2/T3 三档赔付
4. One-Click Bind：保费自动拆到 14 个市场仓位(Polymarket × 5、运价衍生品、Swiss Re 自留、数据费等)

### 三 | 黑客松初体验

此次黑客松的整体体验还是很好的，能够感受到主办方的用心（包括但不限于软硬件支持，住宿保障，志愿服务，午饭茶歇等），比赛氛围也很好，选手们都很热情，经常能看到大家围在一起讨论 idea，互相帮忙看项目、解决问题

会场上有些团队的 idea 是相似的，有的团队只是停在一个简陋的demo阶段，而有的团队已经把这个想法往下做了好几层，我觉得 idea 本身在这种活动里并不稀缺，重要的是谁做得更深、想得更细

另一方面，赛前，我以为黑客松是奋战两天三夜、拼命写代码，但整个比赛下来，我觉得现在的黑客松可能更多的是和不同的人 social、分享 idea，寻找投资人、合作伙伴，或是已有落地项目的人来增加其产品曝光度；从 0 开始团队也有不少，但大多只能做出一个 demo，所以最后评委打分也（部分）靠你的演讲能力、应变能力、介绍PPT/视频的质量，但最主要的还是项目的新颖度和市场前景

所以这次活动学到的东西，可能不主要来自"比赛"这件事本身，而更多是来自和不同人的接触，看到他们的状态、倾听他们的经历......

### 四 | 个人反思
最后写一些整体的感受

我注意到不少92学校的同学，看起来“做事不紧不慢”，有点懒散（后来觉得**松弛**这个表达更好），时间观念貌似也不是很强，迟到像是家常便饭一样（不知道这是不是我的偏见）

但他/她们并不是书呆子那样，除了学习其他什么都不做/不会的状态，反而觉得他们比我“会玩”得多——在黑客松现场，有学生 rapper 在活动开始时唱歌活跃气氛，主办方在开发的第二天晚上放歌提神，现场很多人都嗨了起来，像蹦迪现场，但在我(几乎没见过这种场面的人)看来是不明所以的燃起来......
我并不知道为什么要这样做，这也许会让思维更活跃发散吗；

另外还想补一句，他们真的是高精力人群，能够从黑客松中快速恢复过来，而我早早就歇菜了

他们在花钱这件事上不太吝啬，但也很会琢磨怎么赚钱，会做各种尝试。而且他们大多不是单打独斗，可能是与朋友一同参加比赛，氛围很不错，互相之间也会讨论学业、就业、创业，分享经验，他们其中还不乏正在创业/有创业想法的人

这种环境我其实挺少接触到的，大部分时候是自己一个人琢磨。之前一直以为我们之间的差距是能力，我只需要再努力些，往学习上投入更多的时间，但这次让我意识到，可能不完全是。比如说，我也能刷到很多人在做各种事的视频/帖子，但"刷到"和"身边有人在做"是不一样的——前者让我知道这件事存在，后者才会让我觉得"我也可以试一下"。再比如，我自己做东西卡住的时候，只能自己琢磨，琢磨不出来我大概率就放弃了，他们身边有一群在做类似事情的人，卡住了可能和朋友讨论一下就能解决

还有就是，他们身边有人在做厉害的事，这种压力是具体的、看得见的；我没有这种同侪，标准只能跟自己比，很容易觉得"我已经挺努力了"

---

除了环境之外，这次黑客松还让我意识到一件事，我现在的状态，大概更多是靠自己的意志力——"我得好好学，我得学会这个技能"，好像一直都没想清楚"我到底在为什么学"。
也许我一直在错误的层面上找"目标"——我以为我要找的是一个 destination，但真正缺的可能不是"目标"，是一种"在做一件具体事情"的状态。

"目标"是未来的某个点，是想象中的奖励；"状态"是当下的姿态，是做的过程本身能给你反馈。**有目标但没状态，人会拖延**；而 **有状态但不一定有清晰的远期目标时，人反而能动起来**——港大那个同学最早学嵌入式的时候，做的第一个作品是给女朋友的冰淇淋机，他说那时候不是为了"成为嵌入式专家"才学的，而是在做一件具体的、有反馈的事；后来创业开公司，"员工不会的我得会"对他来说也不是一个抽象的目标，是一个具体的、被推着学的状态。

我现在的"要好好学cs"，它是一个目标，但我没有一件具体在做的事来承载它。所以我学完之后会感觉"学了但没内化"，因为**学习本身不构成一个有反馈的状态**，必须有"我在用它做什么"才会有反馈，才会真的留下东西。
（后来和我哥聊起这个，他举了一个我觉得挺恰当的例子：日本有个马拉松冠军被问怎么夺冠，他说他每次的目标都是前面那棵树，跑到那棵树之后再以下一棵为目标，而不是想着 42 公里之外的终点。我觉得'状态'差不多就是这个意思，不是不要远期目标，而是**在每个当下都应该有一个具体的、看得见的'下一棵树'**）

写到这里，我又想到队友说的话，"既然你是就业向的，你可以学一些招聘要求里有的东西，多做几段实习"，我当时听到的时候，觉得这个建议有点功利。但仔细想一下，她和港大同学说的应该是同一件事的两种说法（港大同学的锚点是"我开了公司，公司需要我会"，队友建议的锚点是"我要找工作，我需要学会"），两个都是"被现实推着学"的具体形式，只是推力的来源不同。

这次黑客松我看到了很多，也意识到有很多事情自己还没想清楚，可能需要慢慢摸索才能有答案......
