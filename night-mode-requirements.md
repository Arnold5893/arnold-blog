##  暗色模式实现需求

### 一、主题色彩配置

在 Tailwind 配置或全局 CSS 变量中新增暗色模式 token：

/* 暗色模式变量 */
--color-bg-primary: #0F0E17;        /* 主背景：深蓝黑 */
--color-bg-secondary: #1A1A2E;      /* 卡片/容器背景 */
--color-bg-tertiary: #22223a;       /* 悬浮/高亮层 */
--color-accent: #D4919F;            /* Accent：微灰樱花粉 */
--color-accent-subtle: #2a1520;     /* Accent 淡色背景（标签用） */
--color-text-primary: #FFFAF8;      /* 主文字 */
--color-text-secondary: #a0a0c0;    /* 次级文字 */
--color-text-muted: #7070a0;        /* 弱化文字（日期、meta） */
--color-border: #2a2a40;            /* 边框 */
```css

### 二、明暗切换按钮

在导航栏右上角添加切换按钮，要求：

- 默认跟随系统 `prefers-color-scheme`，用户手动切换后写入 `localStorage` 记忆偏好
- 切换时整个页面颜色过渡动画：`transition: background-color 300ms ease, color 300ms ease`

### 三、星空背景动画组件

创建一个独立组件 `StarryBackground.astro`，在暗色模式下渲染到页面背景层（`position: fixed; inset: 0; z-index: 0; pointer-events: none`），由三层叠加组成：

**第一层：繁星闪烁（全页打底）**

- 用 JavaScript 在 canvas 上随机生成 **120 颗星点**，分布在整个视口
- 星点分两种：白色小星（占 70%，radius 0.6～1.2px）、Accent 粉星（占 30%，radius 0.8～1.5px，颜色 `#D4919F`）
- 每颗星独立的 opacity 闪烁动画，周期 1.5s ～ 3.5s 随机，形成自然的星空呼吸感
- 移动端将数量减少至 **60 颗**，避免性能问题

**第二层：流星划过（偶发动效）**

- 每隔 **4～8 秒**随机触发一次流星
- 流星从页面右上角区域随机起点出发，向左下方以约 **45°** 角划过
- 流星本体：长度约 `120px`，线宽 `1.5px`，颜色为白色到透明的线性渐变，头部带一个 `2px` 的亮点
- 偶发第二道细流星（颜色改用 `#D4919F`，线宽 `1px`），与主流星错开 `0.3s` 出现
- 整个动画持续 `0.8s`，结束后消失，等待下一次随机触发

**第三层：星座连线（局部装饰）**

- 在页面**右下角**绘制一组固定的星座连线图案，大小约 `200×150px`
- 包含 **5～6 个星点**，用细线（opacity `0.2`，颜色 `#D4919F`）相连，构成一个简单星座轮廓（可参考大熊座或猎户座的简化版）
- 星点本身参与闪烁动画（同第一层规则）
- 在移动端将星座移到页面**底部居中**，缩小至 `60%` 大小
```

svg（流星）   ← 中层  
svg（星座）   ← 顶层装饰

### 四、暗色模式下各页面适配

**全局**

- 所有白色卡片 → `#1A1A2E`
- 所有边框 → `#2a2a40`
- `<code>` 代码块背景 → `#12111f`，文字颜色适配

**导航栏**

- 背景改为 `#0F0E17`，加 `border-bottom: 0.5px solid #2a2a40`
- Logo / 当前页高亮颜色 → `#D4919F`

**文章卡片**

- 卡片背景 `#1A1A2E`，hover 时背景变为 `#22223a`
- 分类标签：背景 `#2a1520`，文字 `#D4919F`
- 封面图保持原始颜色，不做任何滤镜处理

**Giscus 评论区**

- 在初始化 Giscus 时根据当前主题传入参数：`data-theme="dark"` 或 `data-theme="light"`
- 用户切换主题时动态更新 Giscus iframe 的 theme 参数

### 五、注意事项

- 星空背景组件**仅在暗色模式下显示**，白天模式下完全隐藏（`display: none`）
- 白天模式的樱花花瓣飘落动画在暗色模式下也**完全隐藏**，两套动效互斥
- 星空 canvas 需监听 `resize` 事件，窗口大小变化时重新初始化星点位置

### 六、立绘图暗色模式融合需求

#### 问题描述
首页 Hero 区的动漫立绘图片（jpg，有浅色背景）在暗色模式下与周围深蓝黑背景 #0F0E17 产生割裂感，需要让图片自然融入暗色环境。
#### 解决方案：CSS mask 边缘渐变融合
给立绘图片的容器元素在暗色模式下添加 mask-image，让图片边缘向四周自然消融。
#### 核心 CSS：
/* 暗色模式下生效 */
.dark .hero-avatar-wrapper {
  -webkit-mask-image: radial-gradient(
    ellipse 75% 80% at 55% 45%,
    black 40%,
    rgba(0, 0, 0, 0.6) 65%,
    transparent 100%
  );
  mask-image: radial-gradient(
    ellipse 75% 80% at 55% 45%,
    black 40%,
    rgba(0, 0, 0, 0.6) 65%,
    transparent 100%
  );
}

#### 参数说明

ellipse 75% 80%：椭圆形遮罩，横向略窄、纵向略高，贴合立绘人物的轮廓比例
at 55% 45%：焦点略偏右上方，对应图片中人物脸部的位置，确保脸部最清晰
40% 完全不透明区：人物主体核心区域保持 100% 原色
65% 半透明过渡区：边缘开始柔和消融
100% 完全透明：图片四周彻底融入背景，白色边缘消失

#### 注意事项

白天模式不加任何 mask，仅在 .dark 类下生效，两种模式完全独立
如果立绘容器本身有 overflow: hidden 或 border-radius，需确保 mask 加在最外层包裹 div 上而不是 <img> 标签上
渐变中心点 at 55% 45% 需根据实际图片中人物脸部位置微调，目标是让脸部处于最清晰的核心区
如果融合效果不够自然，可以将过渡区的 0.6 调低至 0.3，让边缘消融更快更彻底
